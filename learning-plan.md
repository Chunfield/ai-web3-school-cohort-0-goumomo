# Learning Plan

## Goal

Build enough AI x Web3 understanding and project proof-of-work to participate in hackathons with confidence.

## Strategy

- Use AI strengths to accelerate reading, note-taking, and prototyping.
- Prioritize Web3 fundamentals first because that is the current weakest area.
- Convert concepts into small experiments and project fragments every week.

## Phase 1: Foundation Ramp

Target: 2 weeks

- AI baseline review: LLM, Prompt, Context, RAG, Agent
- Web3 baseline: Network, Cryptography, Wallet, Smart Contract
- Bridge entry: Chain-aware Context, Web3 Tool Use, Agent Workflow
- Output:
  - 5 to 8 concept notes
  - 2 beginner experiments
  - 1 hackathon project problem statement

## Phase 2: Build Readiness

Target: 2 to 3 weeks

- Account Abstraction and wallet permission models
- Indexing, Oracle, and Security basics
- Simple onchain + AI workflow prototype
- Output:
  - 1 end-to-end demo repo or experiment
  - 1 architecture note
  - 1 risk checklist for agent + wallet flows

## Phase 3: Hackathon Execution

Target: ongoing

- Narrow to one project direction
- Build MVP, demo script, and submission assets
- Record tradeoffs, blockers, and iteration notes
- Output:
  - MVP demo
  - submission draft
  - retrospective

## Weekly Focus

### Week 1

- Read Handbook overview plus AI basics: LLM, Prompt, Context, Agent
- Read Web3 basics: Network, Wallet
- Start a glossary in `tasks/`

### Week 2

- Read Cryptography and Smart Contract
- Read Chain-aware Context and Web3 Tool Use
- Build one toy flow that maps an AI action to a Web3 tool call

### Week 3

- Read Agent Workflow, Wallet / Permission, Security
- Draft a hackathon project brief in `hackathon/`
- Build a small prototype path

## Daily Rhythm

- 60 to 90 min reading and note-making
- 60 to 90 min experiment or project work
- 20 min daily note and check-in draft
- 10 min feedback capture for unclear docs or blockers

## Success Signals

- Can explain wallets, signatures, contracts, and agent workflow in plain language.
- Can sketch a safe human-in-the-loop AI x Web3 product flow.
- Can ship at least one small public proof-of-work each week.
